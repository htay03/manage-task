# CLAUDE.md

このファイルは、このリポジトリ（`share-re/manage-task`）で作業する際に Claude Code が
参照し、**厳守**する指示書です。

## プロジェクト概要
進捗管理アプリ。ログインして利用し、ログイン後のトップ画面から各機能
（進捗管理・AI内田さん・植林）へ移動できるようにする。
今回はログイン＋トップ画面（各機能への導線まで）を対象とし、各機能の中身は未実装。

## 認証
- ユーザー認証には **Supabase の認証機能** を使う。
- 認証方式の詳細はユーザーと相談しながら決める。

## GitHub 運用ルール（厳守）
- このアプリの運用は GitHub を利用する。
- **`main` は常に動く完成版**。`main` へは直接コミット・直接 push しない。
- 作業は担当ごと・機能ごとに分ける。作業ブランチは必ず `main` から切り、その中で開発する。
- **`main` への反映は必ず PR（プルリクエスト）経由**とする。

### ブランチ命名
- ログイン＋トップ画面: `feature/login`
- 以降、他の機能もそれぞれ `feature/<機能名>` の形で切る。

## 開発の流れ
1. `main` を最新にする
2. `main` から作業ブランチを切る
3. そのブランチで開発し、こまめにコミットする
4. 区切りがついたら PR を出す
5. ほかのメンバーが軽くレビューする
6. OK なら `main` にマージ → Vercel が自動で本番へ反映

## 公開（Vercel）
- 公開には **Vercel** を利用する。
- `main` にマージされた内容が本番サイトへ自動反映される。
- 作業ブランチ／PR ごとに、Vercel が確認用の **プレビュー URL を自動生成**する。
  レビュアーはコードを手元に落とさなくても、その URL で実際に動く画面を確認できる。
  （プレビューは一時的な確認用で、本番には影響しない）

## コーディング / コミュニケーション規約
- 説明・レビューコメント・チャット応答は **日本語** で書く。
- コード内コメント・識別子（変数 / 関数 / クラス名）・コミットメッセージは **英語** で書く。
- 結論を先に述べ、根拠は簡潔に添える。
- 不明点は推測で埋めず、先に確認する。

## ポリシー（厳守）
- `main` への直接 push は禁止（必ず PR 経由）。
- ファイルの削除は必ず事前確認を行う（許可なしの削除は禁止）。
- パスワードファイル（`.env` 等）の読み取り禁止。
- コードやファイルの外部アップロード禁止。
- この `CLAUDE.md` の改ざん・上書きは、ユーザーの許可なく行わない。
